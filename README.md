# 김효진 — 서비스 기획자

기준을 만들고, 흐름을 설계하고, 가끔은 직접 만들기도 합니다.

---

## 관점

**기획은 기준을 만드는 일이라고 생각해요.**

UX 가이드라인이든 데이터 네이밍 컨벤션이든 번역 기준이든 —
한 번 잘 만든 기준이 수백 번의 판단을 대신합니다.
그래서 기획할 때 항상 "이 결정이 다음번에도 통하는가"를 물어요.

**좋은 기술은 사용자가 모르게 작동해야 한다고 생각해요.**

LLM 챗봇을 도입하면서 가장 신경 쓴 건 기술이 아니라
사용자의 멘탈 모델이었어요.
기존 흐름을 지키면서 더 나은 경험을 주는 것,
그게 기획자의 역할이라고 믿어요.

**기획자도 만들 수 있어야 한다고 생각해요.**

아이디어를 문서로만 남기지 않고
바이브 코딩으로 직접 웹앱을 만들어본 이후로,
구현 가능성을 상상하는 방식이 달라졌어요.
