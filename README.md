# 개척하는 기획자, 김효진입니다.

구조를 만들고, 흐름을 설계하고, 가끔은 직접 서비스를 만드니다.

---

## Story

### **서비스 기획은 관점을 만드는 일이라고 생각합니다.**

한 번 잘 만든 사고의 틀이 중요하다고 믿습니다.
그래서 UX 가이드라인을 만들었고, 데이터 네이밍 컨벤션을 수립했고, 제품의 다국어 번역 체계를 중복과 누락 없이 체계적으로 설립했습니다.

### **좋은 혁신은 사용자가 모르게 시작되어야 한다고 믿습니다.**

LLM 챗봇을 도입하면서도, 사내  협업 툴을 만들면서도
새로움 대신 익숙함을 최우선으로 신경썼습니다.
남길 것은 남기면서 더 나은 경험을 주는 것이 기획자의 역할이라고 믿습니다.

### **기획자도 만들 수 있어야 한다고 생각합니다.**

바이브 코딩으로 직접 웹앱을 만들어본 이후로 더 큰 가능성을 꿈꿉니다.
아이디어를 문서로만 남기지 않고, 어떻게든 실현시키고 시도하는 사람입니다.
